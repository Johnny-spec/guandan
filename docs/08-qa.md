## §4 Socket Integration Test 设计

`socket.e2e.test.ts` 用真实 `NestFactory.create` + `app.listen(0)`（随机端口）+ 真实 `socket.io-client`：

1. 创建房间 → `room:create` ack。
2. 加 3 个 bots（easy / normal / hard） → `bot:add` × 3。
3. `game:start` → bot 调度器自动接管 3 个 AI 座位。
4. 人类轮 Alice：尝试出最小 cardId，失败则 `pass`。
5. 断言：plays > 20（验证 socket+bot 全链路通路）；本地常稳定跑到 `game:finished`。

未来要增的 case（Phase 2 TODO）：
- [ ] 断线重连：客户端 `socket.disconnect()` → 重新 `io()` → `room:resume`，验证 `state` 恢复。
- [ ] 跨节点：用 `@socket.io/redis-adapter` + 2 个 game-server 实例，验证消息扩散。
- [ ] 反作弊：人类客户端伪造 `game:play` 携带不在手里的牌 → 必须 `ok:false`。

## §5 压测脚本

`tests/load/socket-load.mjs`：

- 启动 `USERS` 个并发用户，每人持续在 `DURATION` 秒内反复打局；
- 收集所有 ack 的 RTT；
- 汇总 p50 / p95 / p99 / max，若 `p95 > 500ms` 或有失败连接则 exit 1。

CI 中暂不跑（需要外部 game-server）。建议接入 Phase 3：用 GitHub Actions self-hosted runner 拉起 docker-compose，再跑 30s × 50 用户的小型 perf gate。

未来可替换为 **k6 / artillery** 写 yaml 脚本，但当前自研脚本零依赖（仅 `socket.io-client`），更易迭代。

## §6 Playwright E2E

- 仅 Chromium，CI retry=2，本地 retry=0；
- 自动 `pnpm --filter teams-tab dev` 拉起前端；可通过 `E2E_BASE_URL` 指向已部署环境；
- Trace 仅在重试时采集（保持 CI 产物体积可控）；
- 失败自动截图。

## §7 Mock Microsoft Graph

`tests/mock/teams-graph.mjs` 替身：

| Route | 用途 |
| --- | --- |
| `GET /v1.0/me` | Teams SDK 取个人资料 |

## §9 测试金字塔目标

```
                ▲  Playwright E2E (2 → Phase 2 扩到 ~20)
               ▲▲  Socket Integration (1 → 扩 ~10：重连/扩容/反作弊)
              ▲▲▲▲ Vitest Unit (78 现状 → Phase 2 目标 200+)
```

- 单测：纯函数（规则引擎）/ 服务（room、session、auth、bot strategy）。
- 集成：真实 NestApp + Socket.IO，不 Mock 网络层。
- E2E：浏览器视角，覆盖关键用户旅程（登录 / 大厅 / 房间 / 一局完整流程）。
- 压测 / 性能：本地 + perf 环境，绑定 SLO（p95 < 500ms / 错误率 < 0.1%）。
